# c++ server Note

### 1.epoll

#### 1.1 概念

```
    epoll是Linux中的一种I/O多路复用技术，它提供了一种高效的方式来监听多个文件描述符的状态变化
```

#### 1.2 触发模式

```
LT（Level Trigger）水平触发
    在LT触发模式下，当文件描述符的状态发生变化时，epoll_wait()函数会返回相应的事件。无论事件是可读还是可写，都会被触发并返回。

    优点：LT触发模式简单易用，适用于读/写事件的处理。

    缺点：在某些情况下，如果文件描述符的状态频繁变化，会导致epoll_wait()函数频繁返回，进而导致系统负载增加。

ET（Edge Trigger）边缘触发
    在ET触发模式下，只有当文件描述符由不可读/不可写状态变为可读/可写状态时，epoll_wait()函数才会返回相应的事件。换句话说，只有当状态发生变化时才会触发事件。

    优点：ET触发模式可以避免在文件描述符状态频繁变化时导致系统负载增加的问题。

    缺点：对于需要同时处理读/写事件的应用程序，ET触发模式可能不够灵活。
```
